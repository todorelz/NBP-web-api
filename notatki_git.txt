Kroki 

reopzytorium lokalne git
1. wchodzę do dodelowego katalogu i tworzę lokalne repozytorium: np
    $ git init
2. tworzę plik .gitignore, w którym wpisuję nazy plików i katalogów, ktorych nie chę śledzić:
    na pewno folderu ze środowiskiem wirtualnym, wszytskie inne pliki, którymi się nie chcę 
    dzielic a są potrzebne mi w projekcie (np notatki)
3. sprawdzanie statusu:
    git statusu
4. dodanie zmian do staging area 
    git add [nazwa pliku]
    dodanie wszystkich:
    git add .
5. zakomitowanie zmian  
    git commit -m "jakaś wiadomość"
6. tworzę zdalne repozytorium na github
7. podpięcie zdalnego repo do lokalnego
    git remote add origin ....... #skopiowane z github , pamiętać o ssh np:
    git remote add origin git@github.com:todorelz/NBP-web-api.git
8. wypchnięcie zmian/commitów (push) do repozytorium zdlanego
    git push origin master

TWORZENIE WŁASNEGO MODUŁU W PYTHONIE
1. w projkecie utwórz folder (np "utils")
2. w tym folderze dodajesz plik: "__init__.py"
3. dodaj srypt pythonowy w którym będą zdefiniowane funkcje (np "exchanege_rates.py")
    skrypt Python, w którym zapsuję funkcje to MODUŁ i ZAWSZE musi być rozszerzenie .py (nie może być ipynb)
    importy: 
        - .py --> .py
        - .py --> .ipynb
4. importuję potrzebną funkcje z pliku z funkcjami np:
    from exchanege_rates import get_latest_exchanege_rate
    jeżeli importuję z pliku na tym samym poziomie przed nazwą pliku trzeba podać "." np
    będąc w exchanege_rates.py napiszę:
    from .consts import SUPPORTED_CURRENCY_CODES
